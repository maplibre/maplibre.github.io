### Terrain3D: Call for Funding

Support for Terrain3D is one of the [most upvoted](https://github.com/maplibre/maplibre-native/issues/252) features for MapLibre Native. While several, (mostly) vibe coded prototypes have been developed, delivering a production-grade implementation is something that requires a significant investment in graphics engineering and expertise.

We are pleased to announce significant progress! We have identified a company that is willing to commit substantial funding to this initiative. However, **they are seeking an additional co-funding partner to bring this feature to life**.

<Announcement>
  If your organization is interested in supporting this development, please
  reach out to us at team@maplibre.org. We are also open to connecting with
  individual developers who are interested in contributing to this effort.
</Announcement>

To clarify, MapLibre (the organization) is not directly funding Terrain3D for MapLibre Native at this stage. Instead we are facilitating connections between interested parties (companies, developers). MapLibre will provide maintainer support through consensus building, quality control, CI/CD, communication and release management.

We have created a dedicated branch `feature/terrain-3d` for continued work on the initial prototype. You can follow the progress in [this PR](https://github.com/maplibre/maplibre-native/pull/4190).

## 🌐 MapLibre GL JS

We had a great month with 5 new releases: [v5.20.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.20.0), [v5.20.1](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.20.1), [v5.20.2](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.20.2), [v5.21.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.21.0), [v5.21.1](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.21.1).

✨ Some of the highlights of this month include:

- We introduced updateable features using [`geojson-vt`](https://maplibre.org/geojson-vt/) which improved the geojson update performance significantly. While this initially caused a few nasty bugs which were quickly resolved, thanks to [Ryan Scully](https://github.com/wayofthefuture), the results are worth it. The latest version is reportedly 7x faster than when we began GeoJSON optimizations a few months ago. Many thanks to everyone who contributed code, testing, and bug reports!
- Memory optimizations and removal of legacy code supporting outdated browsers, mainly lead by [Johan](https://github.com/johanrd).
- Improved ETag handling added by [Rivka Matan](https://github.com/rivkamatan) and [Ryan Scully](https://github.com/wayofthefuture) stops the map from re-downloading unchanged data, saving bandwidth and speeding up load times.
- [Resampling](https://maplibre.org/maplibre-style-spec/layers/#resampling) support available now, added by [Lars Maxfield](https://github.com/larsmaxfield). This option lets users turn off smoothing to see raw, pixel-accurate terrain data instead of a blended or blurry image
- 3D tiles example added by [Hang](https://github.com/hh-hang)

Last month we mentioned starting pre-releases for Version 6. However due to the high volume of contributions to the current branch, we have decided to continue with Version 5 for now. We still aim to begin the Version 6 transition in the coming months.

## 🧩 MapLibre Tile (MLT)

The [MapLibre Tile (MLT) project](https://maplibre.org/news/2026-01-23-mlt-release/) has seen significant progress this month. The Rust implementation is rapidly maturing; we have introduced a comprehensive suite of synthetic tests to ensure stability across edge cases.

Beyond just stability, we have achieved full decoding and expanded partial encoding support, bringing us closer to a production-grade toolkit. This progress is a major step toward making MLT a viable, high-performance alternative to MVT, offering better compression and faster parsing for large-scale geospatial data.

You can follow the development and check out the [MLT Specification](https://github.com/maplibre/maplibre-tile-spec) on GitHub.

## 🦀 Martin

This month, like the last month, has been focused on implementing a production-grade MLT encoder/decoder, so there has not been much movement regarding Martin.

We fixed a startup crash that occurred when `ST_Extent` returns a `LineString` instead of `Polygon` , and added [ZSTD support](https://facebook.github.io/zstd/) thanks to [@nuts-rice](https://github.com/nuts-rice).
We also further reworked [our new documentation site](https://maplibre.org/martin/) and ironed out a few of the remaining inconsistencies.

## ✨ Community spotlight

We are tracking an exciting development in the rendering world: [the Slug algorithm is now out-of-patent](https://terathon.com/blog/decade-slug.html).

Most maps currently draw text using pictures of letters, which can look fuzzy when you zoom in close to the map view. The Slug algorithm replaces these pictures with mathematical blueprints, allowing labels to remain perfectly crisp at any angle or scale.
