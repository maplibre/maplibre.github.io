## üì± MapLibre Native

**Faster, Synchronous GeoJSON updates:** The latest releases for MapLibre iOS and Android allow adding GeoJSON sources which use synchronous updates. This is especially useful for time-sensitive GeoJSON layers such as the location puck (the internal location indicator layer on Android has been updated to use a synchronous GeoJSON layer). Please try it out and share your experiences. Thanks [Alex Cristici](https://github.com/alexcristici) for implementing this functionality.

**Terrain 3D support:** [Jesse Crocker](https://github.com/jessecrocker) and [Nathan Olson](https://github.com/NathanMOlson) are cooking up a Terrain 3D implementation for MapLibre Native. Check out the work in progress [here](https://github.com/JesseCrocker/maplibre-native/pull/2) and [here](https://github.com/NathanMOlson/maplibre-native/pull/1).

**Enhanced Hillshade & Color-Relief Layers:** [Andrew Calcutt](https://github.com/acalcutt) is working on a PR to [update Hillshade Algorithms and add Color-Relief Layer support](https://github.com/maplibre/maplibre-native/pull/3965). Please have a look, try it out if you can and give his PR a +1!

**Android Transofrm Updates:** We are still gathering feedback on [an Android pre-release](https://github.com/maplibre/maplibre-native/releases/tag/android-v12.3.0-pre0) that enables icon scaling with offsets and makes changes to the transform implementation to allow for concurrent animations. If you're affected by these changes, please try it and help ensure there are no regressions in your app.

## üåê MapLibre GL JS

Version [5.14.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.14.0) and [5.15.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.15.0) are now available! These releases include several bug fixes related to terrain, canvas source, abort errors, GeoJSON handling and general packaging improvements.
Another notable addition is support for [`global-state`](https://maplibre.org/maplibre-style-spec/expressions/#global-state) in expression visibility.

Many thanks to all the contributors in these releases: [Lucas Wojciechowski](https://github.com/lucaswoj), [David E Disch](https://github.com/Auspicus), [Ryan](https://github.com/wayofthefuture), [Mary](https://github.com/mmc1718), [Andrew Dassonville](https://github.com/andrewda), [Omkarthipparthi](https://github.com/Omkarthipparthi), [Lars Maxfield](https://github.com/larsmaxfield)

**GeoJSON Improvements**

Additionally, [geojson-vt](https://github.com/maplibre/geojson-vt) has been onboarded to the MapLibre GitHub organisation to support incremental tile updates for GeoJSON. We are also researching solutions to some long-standing issues in GeoJSON usage, such as handling nested and undefined properties.

**MLT TypeScript**

There's [an open PR to support decoding of FastPFor](https://github.com/maplibre/maplibre-tile-spec/pull/714), which is the final missing piece for version 1.0 of the tile specification. In parallel, ongoing work is focused on performance optimizations, along with a strong emphasis on unit tests to ensure the codebase is well covered.

**NPM & Tooling**

We've moved all our packages to trusted publishing and aligned the CI workflows to make the changelog and release process consistent across most packages.

## üß© Martin

Development slowed down a bit over the past month due to the holiday season, but we still managed to ship a handful of useful improvements in the [1.1.0](https://github.com/maplibre/martin/releases/tag/martin-v1.1.0) and [1.2.0](https://github.com/maplibre/martin/releases/tag/martin-v1.2.0) releases.

Highlights include better support for automation, more flexible error handling, and a few targeted fixes and performance tweaks:

- The `mbtiles summary` command now supports `--format [json | json-pretty | text]`, making it easier to integrate with scripts and other automated tooling (by [Yuri Astrakhan](https://github.com/nyurik)).
- A new `on_invalid: warn` option allows invalid sources to emit warnings instead of aborting execution, giving users more control over failure behavior (by [Gabe Schine](https://github.com/gabeschine)).
- Fixed an issue where PostgreSQL sources with `auto_publish: false` would incorrectly warn about all tables and functions; warnings are now limited to explicitly configured sources (by [Brian Todt](https://github.com/todtb)).
- `martin-cp` now infers default bounds from configured sources, improving performance when copying from sparse source sets (by [Brian Todt](https://github.com/todtb)).
- Discovery now supports `az://` URL schemes, enabling Azure Blob Storage‚Äìbased workflows (by [Frank Elsinga](https://github.com/CommanderStorm)).

Even with a quieter month, these changes help smooth out common workflows and make Martin easier to integrate into larger systems.

## ‚ú® MapLibre SwiftUI DSL

It's been a few months since our last update, but the [SwiftUI DSL continues to add new features and receive updates](https://github.com/maplibre/swiftui-dsl) to keep pace with the latest MapLibre Native releases as well as updates from Apple.

Some highlights include:

- Support for the latest SwiftSyntax versions and iOS 26
- A half dozen or so new style property bindings
- Moving more properties to be controlled via the environment, which should help clean up a lot of code going forward!

Looking back on the year, we've picked up quite [a few new contributors](https://github.com/maplibre/swiftui-dsl/graphs/contributors?from=1%2F4%2F2025)! While we don't yet know how this correlates to usage, it's reasonable to call it a success. Looking forward to 2026, our process for exposing API bindings appears to be working well, so it's time to consider an automated approach to fill in the rest of the missing bindings. Expect an update in future newsletters, or join us in the `#maplibre-swiftui` channel on Slack.

## ü™∂ MapLibre Flutter

Since the last update in the September newsletter, we've continued to focus on good stability and cross-platform experience for developers. Version [0.3.1](https://pub.dev/packages/maplibre/changelog#031) and [0.3.2](https://github.com/josxha/flutter-maplibre/blob/main/maplibre/CHANGELOG.md#032) have been released, bringing several notable improvements:

- Moving to 100% ffi interop on Android. We are looking forward to achieving the same on iOS soon.
- Support for creating map markers from [Material](https://api.flutter.dev/flutter/material/Icons-class.html), [Cupertino](https://api.flutter.dev/flutter/cupertino/CupertinoIcons-class.html) or any other Flutter [IconData](https://api.flutter.dev/flutter/widgets/IconData-class.html). These icons can be displayed on the map without any delay between Flutter and the MapLibre map. For the future, support for rendering complete widgets as map markers is planned.

Another exciting release, version [0.3.3](https://github.com/josxha/flutter-maplibre/milestone/13), is coming soon. A big thanks to everyone involved in the progress and development throughout 2025!
