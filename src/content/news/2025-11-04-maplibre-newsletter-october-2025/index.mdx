## üåê MapLibre GL JS

We have released two major versions this month: [v5.9.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.9.0) and [5.10.0](https://github.com/maplibre/maplibre-gl-js/releases/tag/v5.10.0).
The major highlights of these versions are the support for both `line-dasharray` and `line-gradient` which looks very nice as can be seen [here](https://maplibre.org/maplibre-gl-js/docs/examples/create-a-gradient-dashed-line-using-an-expression/).

There was also the addition of time control APIs to allow creating videos, and the `transformConstrain` hook ü™ù to allow changing how the view is constrained, for example by allowing more padding to the map for applications that might not need a "classic" map. _(Check out the [GitHub issue](https://github.com/maplibre/maplibre-gl-js/issues/4510) for further details)_.

üôè Thanks Brice Person, Christian Neumann, Lars Maxfield, Federico Tibaldo, Lucas Wojciechowski, Manuel Emeriau, Andrew Dassonville, Ryan Scully, Samarth Otiya, Corado Burcus for your contributions!

We are also working on integrating MLT (MapLibre Tiles) into maplibre-gl-js. We have a [draft PR](https://github.com/maplibre/maplibre-gl-js/pull/6570) which you can start playing with, we are aiming for an initial support in November, and we will continue the journey of improving its performance and capabilities going forward.
The TypeScript MLT decoder package pre-release can be found on [npm](https://www.npmjs.com/package/@maplibre/mlt).

Last but not least we are exploring ways to change how maplibre-gl-js over-scales vector tiles at high zoom levels.
The following two interconnected PRs are a first draft on solving that, but we need more feedback from developers to know if this is the right approach and if this is something that can be useful. The main motivation around these PRs is to solve a Safari crash on mobile devices when there are a lot of lines and a lot of labels in high zoom levels (contours is the obvious example for this).

- https://github.com/maplibre/maplibre-gl-js/pull/6521
- https://github.com/maplibre/maplibre-gl-js/pull/6617

_Martin v0.20.0 - the final step before 1.0 üéâ_

This month in Martin, we have published version v0.20.0

We‚Äôre getting close!

This release marks what we expect to be the **final beta before Martin v1.0**.
The architecture is now locked down, and unless we uncover major bugs, this version will be **re-published as v1.0** within the next week.

Huge thanks to everyone contributing code, reviews, testing, and feedback ‚Äî you make Martin faster, more reliable, and more welcoming ‚ù§Ô∏è

The hallmark features in this release are:

- ‚ö° **Smarter, more flexible cache**
  The cache is now split into multiple parts ‚Äî sprites, fonts, PMTiles directories, and tiles ‚Äî each with its own configurable limit.

- ‚òÅÔ∏è **PMTiles: now for GCP, Azure, and more**

  Martin [now supports the following pmtiles backends](https://maplibre.org/martin/sources-files.html#serving-pmtiles-from-local-file-systems-http-or-object-storage):
  - Google Cloud and Azure object storage
  - Expanded AWS and HTTP backends
  - Local files as before

- üß™ **Experimental: server-side style rendering**

  You can now render MapLibre styles directly on the server to generate raster tiles. Still experimental, but great for early non-production testing.

- Because of the level of maturity, we demoted the `cog` feature to `unstable-cog`. This means that it is not avaliable in the default bundle anymore, as the feature is still evolving.

If all goes smoothly, expect **Martin v1.0.0 final** very soon üöÄ
