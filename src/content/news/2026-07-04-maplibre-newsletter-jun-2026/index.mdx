## 📱 MapLibre Native

- Thanks to the efforts of Adrian Cojocaru and Alex Cristici, the [instancing based fill extrusion optimizations](https://github.com/maplibre/maplibre-native/pull/4310) is now implemented for the Vulkan backend as well. If you are using MapLibre Android, this is yet another reason to switch to the Vulkan backend (the default since version 13.0.0).

- Harel Mazor shared a [blog post](https://medium.com/@harel.mazor/putting-mapeak-on-the-dashboard-a6cc73d4f99e) detailing his experiences adding Android Auto and Carplay support to his app using MapLibre Native. He also published a project to bring contours to MapLibre iOS and Android using a Rust-based [HTTP interceptor](https://github.com/Mapeak-com/maplibre-contour-rs).

- Over on Slack, Vladyslav Odobesku created a [discussion thread](https://osmus.slack.com/archives/C04G140P9U6/p1781859352910629) about adding a Kotlin based `CustomVectorTileProvider` that would be a more performant alternative to `CustomGeometrySource` for Android developers.

- [MapLibre Native FFI](https://maplibre.org/maplibre-native-ffi) project now features browser support using the WebGPU backend. Dive into the [Slack discussion thread](https://osmus.slack.com/archives/C02B2CBSNBU/p1782547722335569?thread_ts=1778119369.635259&cid=C02B2CBSNBU) and [the relevant PR](https://github.com/maplibre/maplibre-native-ffi/pull/285) for more details.

- Taku Fukada from our valued sponsor MIERUNE implemented [double resolution rendering for CJK characters](https://github.com/maplibre/maplibre-native/pull/4304). This makes these characters much more legible on mobile screens. MapLibre GL JS has had this behavior for a while already and we're thrilled to bring this crisp rendering to MapLibre Native.
