## 📱 MapLibre Native

**Memory Optimization**: Taking advantage of [instancing](https://en.wikipedia.org/wiki/Geometry_instancing), Alex Christi managed to significantly reduce the memory usage of fill extrusion layers (see [PR](https://github.com/maplibre/maplibre-native/pull/4256)). Right now, this is implemented for the Metal backend, with Vulkan following suit thanks to Adrian Cojocaru's work (see [PR](https://github.com/maplibre/maplibre-native/pull/4310)). If you are a heavy user of fill extrusion layers, we would love to hear your experience.

**Android Renderer**: With Vulkan now being the default renderer in MapLibre Android 13.0.0, more people are trying out the Vulkan backend. MapLibre Android [13.2.0](https://github.com/maplibre/maplibre-native/releases/tag/android-v13.2.0) introduced some Vulkan specific bugfixes.

**HarmonyOS Ecosystem Expansion**: [This MapLibre Native PR](https://github.com/maplibre/maplibre-native/pull/4315) adds support for a new platform, [HarmonyOS](https://en.wikipedia.org/wiki/HarmonyOS), Huawei's commercial OS competing with Android and iOS in China. It is related to [OpenHarmony](https://en.wikipedia.org/wiki/OpenHarmony), an open source project also utilized by initiatives like [Oniro](https://oniroproject.org).

**Cross-Language Bindings**: Separately, [MapLibre Native FFI](https://github.com/maplibre/maplibre-native-ffi) (the experimental C API wrapper for MapLibre Native), has made great progress since onboarding last month. It has officially landed Java, Rust, Zig, and Kotlin Native bindings, while filling out its support matrix across macOS, Windows, and Linux using Vulkan, Metal, and OpenGL.

<p>
  Dave Brophy opened several PRs this month, including an implementation of
  [contour lines](https://github.com/maplibre/maplibre-native/issues/4283) based

  means a feature like this can only be contributed to the core library by
  someone with a strong graphics background and knowledge of MapLibre Native
  rendering internals.
</p>

<Announcement>
  <h4>Code Integrity & AI Policy</h4>
  <p>
    We want the core of MapLibre Native to be stable and maintainable for the
    long term. This means being judicious in accepting pull requests, especially
    those written using an LLM. At the same time, we understand that some of our
    users want to move quickly, and these days that often means using AI to
    write functionality.
  </p>
  <p>
    <strong>Our Solution:</strong> We plan to create a [plugin
    API](https://maplibre.org/roadmap/maplibre-native/plugin-api/) for MapLibre
    Native. This will allow developers to extend the library without changing
    the core codebase. As an example, MapLibre GL JS has a popular [contour
    plugin](https://github.com/onthegomap/maplibre-contour); in the future,
    MapLibre Native might have one as well.
  </p>
</Announcement>

## 🦀 Martin

Nine `martin` [releases](https://github.com/maplibre/martin/releases) shipped across April and May (1.5.0 through 1.10.1), with matching `martin-core` and `mbtiles` versions.
April is folded in here because no changelog went out at the time.

Key highlights include:

**On-the-fly MLT support**: The biggest change is [MLT (MapLibre Tiles)](https://maplibre.org/news/2026-01-23-mlt-release/) support.
Martin now transcodes MVT to MLT (and back) on the fly, picking the format from the client's `Accept` header.
On our test fixture, MLT is about 39% smaller on the wire and about 12% faster to serve from cache due to that. Checkout the [full guide](https://github.com/maplibre/martin/blob/main/docs/content/using-guides/mlt.md) if you want to dig in.

**Server-side raster rendering**: Now stable and on by default on Linux builds via MapLibre Native (`styles: rendering: true`). This closes a massive milestone [#978](https://github.com/maplibre/martin/issues/978) open since October 2023.

**Live source reloading**: MBTiles directories hot-reload as of April, COG and PMTiles followed in May, and PostgreSQL groundwork is in place for the PG reload work that comes next.

## ✨ Community Spotlight

**OMT Router**: Built by Abel, this fast, zero‑backend library builds routing graphs straight from OpenMapTiles vector tiles to compute optimal routes and isolines entirely in the browser.
Dive into the [source code](https://github.com/AbelVM/omt-router) or test it yourself on the [live interactive demo](https://abelvm.github.io/omt-router/example/).

**QGIS2VectorTiles**: Returning with a major upgrade since its last feature, [QGIS2VectorTiles](https://gallpeters.github.io/QGIS2VectorTiles/) packs a QGIS project into a lightweight package complete with vector tiles, a matching MapLibre style, and a ready-to-use web viewer.

- Recent updates to the project introduces native MBTiles export, imagery background layers, alpha channel color support, and a redesigned multithreaded export pipeline utilizing the GDAL CLI for faster tile generation. It also vastly expands style conversion (adding symbol-placement, text-variable-anchor, and interactive pop-ups) alongside a brand-new OpenLayers web viewer option.
