Adopting the [OGC API - Tiles](https://ogcapi.ogc.org/tiles/) standard and exposing Martin's sources through [STAC](https://stacspec.org/) catalogs would make Martin a first-class citizen in the OGC-standardised stack and the Earth-observation tooling that has standardised on STAC for catalog discovery.

Martin today speaks TileJSON and XYZ - well-suited to the MapLibre, Mapbox-style, and OpenLayers world, but missing the open standards that government data portals and scientific data infrastructures expect to consume.

This work is on the roadmap, but it is **not actively prioritised**.
The conformance surface is large, and current contributor attention is on other features.
This entry needs a **champion** before it will move forward:
someone with a concrete deployment that requires OGC API Tiles or STAC, willing to drive the design, the implementation, and the long tail of conformance testing.
Without one, it stays parked here.

If that's you, please reach out via [maplibre/martin](https://github.com/maplibre/martin) or the MapLibre Slack.
